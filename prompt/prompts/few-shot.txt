You are going to receive a Text. Transform it to a knowledge graph using RDF/Turtle syntax.
Return only the RDF/Turtle output. Do not write explanations, notes, headings, or markdown.
Every subject block must end with a period.
Literal labels must use exactly one pair of double quotes, never doubled quotes.
Use only the declared prefixes below. Do not use undeclared prefixes such as schema: or ex:.
Use the exact Wikidata resource `wd:Q...` for a recognizable real-world entity when you know its QID confidently. The QID must appear as the subject or object of relationship triples, not as a label-only literal.
Do not replace a known Wikidata entity with a generated `kg:` resource. Use `kg:` with a snake_case local name only for concepts that have no confidently known QID; never invent a QID.
Whether the resource is `wd:` or `kg:`, use the exact human-readable surface form from the input Text as its rdfs:label. Do not replace the text wording with a canonical or normalized name.
Use kg: resources, classes, and predicates with snake_case local names only when no confident Wikidata QID is available.
Add a language tag to each rdfs:label literal.
Every entity, concept, class, place, person, organization, event, and object that appears as a subject or object must have an rdfs:label.
Prefer entity-to-entity triples over literal-only descriptions so SPARQL evaluation can traverse to labeled answers.
For type or classification relationships, use kg:is.
When possible, use this predicate vocabulary: kg:is, kg:of, kg:from, kg:in, kg:on, kg:to, kg:with, kg:has_part, kg:part_of, kg:located_in, kg:instance_of.
